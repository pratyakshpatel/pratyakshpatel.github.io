Rotating the Feature Space: A Gentle Journey into Rotation Forests

13 March, 2026

A few years ago I was playing around with tree-based machine learning models. Random forests, gradient boosted trees, extremely randomized trees – these models show up everywhere. They dominate Kaggle competitions, they run many financial prediction systems, and they quietly power a lot of industrial forecasting pipelines. For tabular data especially, tree ensembles are still some of the strongest models we have.

But while experimenting with them, I noticed something slightly odd. Decision trees make decisions in a very particular way. They split data along one feature at a time. In mathematical terms, a typical decision rule inside a tree looks like:

\[ x_j > \tau \]

Here \(x_j\) is one specific feature and \(\tau\) is a threshold. If the feature value is larger than the threshold the tree moves to one branch, otherwise it moves to another.

Geometrically this means something very specific. If we imagine our data points living in a multi–dimensional space, every split made by a decision tree is aligned with one of the coordinate axes. In two dimensions this means the tree can only draw vertical or horizontal lines.

At first this seems harmless. But it turns out this restriction can be surprisingly limiting.

A small geometric thought experiment

Imagine a simple classification problem with two variables \(x_1\) and \(x_2\). Suppose the true decision rule separating two classes is something like

\[ x_1 + x_2 > 0 \]

This rule creates a diagonal boundary across the plane. A linear classifier such as logistic regression would learn this instantly.

But a decision tree cannot split on \(x_1 + x_2\). Instead it must approximate the diagonal boundary using many small axis-aligned rectangles. The resulting model may require many levels of splits to approximate something that is fundamentally very simple.

This observation leads to a fascinating question.

What if we rotated the coordinate system before training the trees?

If we rotate the data by forty five degrees, that diagonal boundary suddenly becomes axis aligned. The tree can then capture it using a single split.

This idea – rotating the feature space so trees can see different perspectives of the data – is the intuition behind an algorithm known as the Rotation Forest.

The birth of Rotation Forests

The Rotation Forest algorithm was introduced by Juan Rodriguez, Ludmila Kuncheva, and Carlos Alonso in a 2006 paper titled “Rotation Forest: A New Classifier Ensemble Method”. The motivation was to improve ensemble diversity while keeping each individual tree strong.

Ensemble learning relies on a simple principle. Instead of training one model, we train many models and combine their predictions. If the models make different mistakes, their errors cancel out when averaged together.

For an ensemble to work well two ingredients are needed. First, each model must be reasonably accurate on its own. Second, the models must be different from each other.

Random forests achieve diversity by sampling different subsets of features and different subsets of training data. Each tree sees a slightly different version of the dataset.

Rotation forests take a completely different approach.

Instead of randomly removing features, they rotate the feature space using principal component analysis.

The core mechanism

Suppose our dataset contains \(d\) features. For example

\[ X = (x_1, x_2, \dots, x_d) \]

The rotation forest algorithm proceeds through a sequence of steps for each tree in the ensemble.

First, the set of features is randomly divided into several subsets. If we have ten features, we might create subsets such as

\[ \{x_1, x_4, x_7\}, \{x_2, x_5\}, \{x_3, x_6, x_8, x_9, x_{10}\} \]

Each subset is treated independently. For each subset we apply principal component analysis.

PCA transforms the original variables into new variables that are linear combinations of the originals. If a subset contains variables \(x_a\) and \(x_b\), PCA might produce new components such as

\[ z_1 = 0.8x_a + 0.6x_b \]

\[ z_2 = -0.6x_a + 0.8x_b \]

These new variables represent rotated axes in the feature space.

The PCA transformations from each subset are then combined into a single rotation matrix. If we denote this matrix by \(R\), the dataset is transformed as

\[ X_{rot} = X R \]

The decision tree is then trained on this rotated dataset.

For the next tree in the ensemble, a different random partition of features is chosen, leading to a different rotation matrix. As a result, every tree sees a unique orientation of the data.

Why rotations help

There are two main reasons why rotations can improve tree ensembles.

The first reason is feature interaction. Many real world relationships involve combinations of variables rather than individual features. Rotations naturally produce linear combinations, allowing trees to split along directions that represent meaningful interactions.

The second reason is ensemble diversity. Because each tree receives a differently rotated dataset, the models become less correlated with each other. Lower correlation among trees improves the effectiveness of averaging predictions.

In fact, the original Rotation Forest paper reported strong improvements over random forests on many classification benchmarks.

A linear algebra perspective

To understand rotations more clearly it helps to look at the underlying linear algebra.

A rotation in \(d\)-dimensional space can be represented by an orthogonal matrix \(R\) satisfying

\[ R^T R = I \]

where \(I\) is the identity matrix.

Multiplying a vector by this matrix rotates it while preserving distances and angles.

In other words

\[ \|Rx\| = \|x\| \]

This property means rotations do not distort the geometry of the data. They simply change the coordinate system.

From the perspective of a decision tree however, the rotated data can look dramatically different. A boundary that was diagonal in the original coordinates may appear perfectly aligned in the rotated space.

The role of PCA

Rotation forests specifically use PCA to generate rotation matrices.

PCA identifies directions in the data with maximum variance. If the covariance matrix of the data is

\[ \Sigma = \frac{1}{n} X^T X \]

then PCA finds eigenvectors of this matrix. These eigenvectors form the principal components.

Projecting data onto these components produces decorrelated features ordered by importance.

The rotation forest algorithm keeps all principal components rather than discarding low variance ones. This ensures that information is not lost during the transformation.

Random rotations instead of PCA

While PCA is a natural way to generate rotations, it raises an interesting question.

Is PCA actually necessary?

After all, the primary purpose of the rotation step is to create diversity among trees. Perhaps any orthogonal transformation could achieve a similar effect.

This leads to the idea of using completely random rotations.

Instead of computing PCA, we generate a random orthogonal matrix \(R\) and transform the dataset using

\[ X_{rot} = X R \]

Such matrices can be constructed by generating a random matrix and applying QR decomposition.

Interestingly, random rotations preserve distances just like PCA rotations. From a geometric standpoint they are equally valid transformations.

Connections to random projection forests

Random rotations are closely related to another family of algorithms known as random projection forests.

In these methods trees are allowed to split along random linear combinations of features. A split may look like

\[ w^T x > \tau \]

where \(w\) is a randomly chosen direction.

This effectively allows the tree to create oblique decision boundaries rather than axis-aligned ones.

Both random projection forests and rotation forests attempt to overcome the geometric limitations of traditional decision trees.

An interesting connection to time series

The idea of rotations becomes particularly intriguing when applied to time series forecasting.

Time series models often convert past observations into lag features such as

\[ (y_{t-1}, y_{t-2}, y_{t-3}, \dots, y_{t-k}) \]

These lag variables tend to be highly correlated. For example yesterday’s temperature is strongly related to today’s temperature.

When PCA is applied to such lag features it often produces components that resemble meaningful temporal patterns. One component may capture overall trend, another may capture short term momentum, and another may capture sudden shocks.

Rotation forests trained on these transformed features can potentially detect patterns that ordinary trees struggle to capture.

Open questions

Despite their elegance, rotation forests remain relatively underexplored compared to other ensemble methods.

Many interesting questions remain open. Does PCA provide significant advantages over random rotations? How do rotations interact with temporal autocorrelation in time series data? Can rotation based ensembles approximate oblique decision trees more efficiently than existing methods?

These questions suggest that the geometry of feature spaces may still hold many surprises.
